Parameter Guide

解

Model Selection

Choose one or more AI models to compare. Each model has different pricing for input and output tokens. Select multiple models to see a side-by-side cost comparison.

Custom Model

Add your own model with custom pricing. Useful for comparing private or fine-tuned models, or testing hypothetical pricing scenarios against standard offerings.

Total Active Users

The total number of registered users in your application. This is the base pool from which daily active users are calculated using the Daily Active % parameter.

Daily Active %

The percentage of total users who actively use the AI features each day. A 25% rate means 1 in 4 users makes at least one API request daily. Typical range: 10-40%.

Requests / User / Day

How many API calls each active user makes per day. This includes every interaction that triggers a model inference — chat messages, completions, summarizations, etc.

Avg Input Tokens / Request

The average number of tokens sent to the model per request. Includes system prompts, user messages, and context. 1 token ≈ 4 characters or ¾ of a word in English.

Avg Output Tokens / Request

The average number of tokens the model generates per response. Longer, more detailed answers consume more output tokens. Output tokens are typically priced higher than input tokens.

Input & Output $/1M Tokens

The cost charged by the provider per 1 million tokens processed. Input pricing covers your prompts; output pricing covers generated responses. These are the standard API rates.
